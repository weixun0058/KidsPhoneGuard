## 一、核心需求与技术挑战分析

### 1.1 核心用户需求

| 用户角色 | 核心诉求 | 痛点 |
|---------|---------|------|
| **家长** | 有效管控、远程监控、灵活调整 | 不想完全禁止但担心沉迷 |
| **儿童** | 公平规则、正向激励、不被"监视感" | 反感强制限制、渴望自主权 |

### 1.2 技术挑战

| 挑战类型 | 具体问题 | 解决思路 |
|---------|---------|---------|
| **权限获取** | Android 对后台限制严格 | 设备管理员权限 + 无障碍服务 |
| **防绕过** | 儿童可能卸载/强制停止 | 设备管理员锁定 + 防卸载机制 |
| **实时监控** | 后台服务保活困难 | 前台服务 + WorkManager 定期唤醒 |
| **跨设备同步** | 家长端与儿童端数据同步 | 云端数据库 + 实时推送 |

### 2.2 应用模式选择

**推荐方案：双应用模式**

| 模式 | 说明 | 优缺点 |
|-----|------|-------|
| **单应用双角色** | 一个APK，登录区分角色 | 实现简单，但儿童可能切换账号 |
| **双应用独立** | 家长端、儿童端分别安装 | ✅ 推荐：职责分离，安全性更高 |

### 3.2 应用访问控制机制

#### 控制级别

| 级别 | 说明 | 行为 |
|-----|------|-----|
| **完全禁止** | 黑名单应用 | 无法启动，立即锁定 |
| **时段限制** | 仅特定时段可用 | 非时段内禁止 |
| **时长限制** | 每日限额 | 超限后禁止 |
| **自由使用** | 白名单应用 | 无限制（如学习类） |

#### 实现机制

```kotlin
sealed class AccessControlRule {
    data class Blocked(val reason: String) : AccessControlRule()
    data class TimeSlotRestricted(
        val allowedSlots: List<TimeSlotRule>
    ) : AccessControlRule()
    data class DurationLimited(
        val dailyLimitMinutes: Int
    ) : AccessControlRule()
    object Unrestricted : AccessControlRule()
}

class AccessControlService {
    fun checkAccess(packageName: String): AccessDecision {
        val rule = getRuleForPackage(packageName)
        return when (rule) {
            is AccessControlRule.Blocked -> 
                AccessDecision.Denied(rule.reason)
            is AccessControlRule.TimeSlotRestricted -> 
                checkTimeSlot(rule)
            is AccessControlRule.DurationLimited -> 
                checkDuration(packageName, rule)
            is AccessControlRule.Unrestricted -> 
                AccessDecision.Allowed
        }
    }
}
```

## 四、技术选型方案

### 4.1 客户端技术栈

| 层级 | 技术选型 | 选型理由 |
|-----|---------|---------|
| **开发语言** | Kotlin | Android官方推荐，协程支持，空安全 |
| **UI框架** | Jetpack Compose | 现代声明式UI，开发效率高，动画丰富 |
| **架构模式** | MVVM + Clean Architecture | 职责分离，可测试性强 |
| **依赖注入** | Hilt | Google官方推荐，编译时检查 |
| **本地数据库** | Room | ORM支持，类型安全，LiveData支持 |
| **网络请求** | Retrofit + OkHttp | 成熟稳定，协程支持 |
| **图片加载** | Coil | Compose友好，协程支持 |
| **异步处理** | Kotlin Coroutines + Flow | 轻量级，结构化并发 |

### 4.2 后端技术栈

| 层级 | 技术选型 | 选型理由 |
|-----|---------|---------|
| **运行时** | Node.js / Python FastAPI | 高并发，实时通信友好 |
| **数据库** | PostgreSQL + Redis | 关系数据 + 缓存组合 |
| **实时通信** | WebSocket / Firebase | 实时推送通知 |
| **认证** | JWT + OAuth2.0 | 安全可靠 |
| **云服务** | Firebase / 阿里云 | 国内访问友好 |

### 6.2 关键安全实现

```kotlin
class SecurityManager(
    private val context: Context,
    private val devicePolicyManager: DevicePolicyManager,
    private val accessibilityManager: AccessibilityManager
) {
    fun checkSecurityStatus(): SecurityStatus {
        return SecurityStatus(
            isDeviceAdminActive = devicePolicyManager.isDeviceAdminActive(),
            isAccessibilityEnabled = accessibilityManager.isEnabled(),
            isOverlayPermissionGranted = Settings.canDrawOverlays(context),
            isNotificationListenerEnabled = isNotificationListenerEnabled(),
            isUsageStatsGranted = isUsageStatsGranted()
        )
    }
    
    fun onSecurityCompromised(compromiseType: CompromiseType) {
        when (compromiseType) {
            CompromiseType.ACCESSIBILITY_DISABLED -> {
                showAccessibilityRecoveryDialog()
                notifyParent("无障碍服务被关闭")
            }
            CompromiseType.DEVICE_ADMIN_REMOVED -> {
                lockDevice()
                notifyParent("设备管理员权限被移除")
            }
            CompromiseType.SERVICE_KILLED -> {
                restartMonitoringService()
                notifyParent("监控服务异常停止")
            }
        }
    }
}
```

### 6.3 防绕过策略详情

| 攻击向量 | 防护措施 | 实现方式 |
|---------|---------|---------|
| 卸载应用 | 设备管理员锁定 | `DevicePolicyManager.lockNow()` |
| 强制停止 | 前台服务 + 自重启 | `startForeground()` + `onTaskRemoved` |
| 关闭无障碍 | 监听设置变更 | 定时检查 + 自动跳转设置 |
| 清除数据 | 设备管理员保护 | 阻止清除数据操作 |
| 安全模式 | 启动时检测 | 检测安全模式并锁定 |
| ADB调试 | 检测USB调试 | 提醒家长关闭开发者选项 |
