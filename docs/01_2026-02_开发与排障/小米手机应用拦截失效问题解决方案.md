# 小米手机应用拦截失效问题完整解决方案

## 问题概述

**现象**：
- 小米手机上应用规则可以添加（应用列表获取正常）
- 但应用无法被锁定或拦截
- 荣耀手机功能正常

**根本原因分析**：

### 1. MIUI后台限制机制
MIUI有严格的省电策略，会杀死后台服务和限制后台活动：
- 后台服务被系统杀死
- 前台服务被限制
- 悬浮窗权限被限制

### 2. MIUI无障碍服务限制
- MIUI可能限制了无障碍服务的事件接收
- 无障碍服务在后台被暂停

### 3. MIUI悬浮窗限制
- 悬浮窗显示被延迟或阻止
- 悬浮窗触摸事件被系统拦截

## 二、MIUI特定解决方案

### 2.1 关键设置步骤（必须执行）

#### 步骤1：设置省电策略为"无限制"

```
设置路径：
设置 → 省电与电池 → 应用智能省电 → 儿童手机守护 → 无限制

或：
设置 → 电池与性能 → 应用智能省电 → 找到"儿童手机守护" → 选择"无限制"

MIUI 14路径：
设置 → 省电与电池 → 右上角设置 → 应用智能省电 → 儿童手机守护 → 无限制
```

#### 步骤2：允许自启动

```
设置路径：
手机管家 → 应用管理 → 权限 → 自启动管理 → 开启"儿童手机守护"

或：
设置 → 应用设置 → 应用管理 → 儿童手机守护 → 自启动 → 允许
```

#### 步骤3：在最近任务中锁定应用

```
操作步骤：
1. 打开最近任务（点击多任务键或手势上滑停顿）
2. 找到"儿童手机守护"的卡片
3. 长按卡片或点击卡片上的锁图标
4. 确保显示锁形图标（表示已锁定）
```

#### 步骤4：开启悬浮窗权限

```
设置路径：
设置 → 应用设置 → 应用管理 → 儿童手机守护 → 权限管理 → 显示悬浮窗 → 始终允许

或：
设置 → 隐私保护 → 保护隐私 → 特殊权限设置 → 显示悬浮窗 → 儿童手机守护 → 允许
```

#### 步骤5：无障碍服务设置

```
设置路径：
设置 → 更多设置 → 无障碍 → 已下载的服务 → 儿童手机守护 → 开启

确保：
□ 服务状态显示为"开启"
□ 没有被系统提示"服务异常"而关闭
```

### 3.1 增强服务保活机制

修改`GuardForegroundService.kt`：

```kotlin
/**
 * 前台服务 - 保活核心
 * MIUI优化版：更强的保活机制
 */
class GuardForegroundService : Service() {

    companion object {
        const val CHANNEL_ID = "guard_channel"
        const val NOTIFICATION_ID = 1001

        fun startService(context: Context) {
            val intent = Intent(context, GuardForegroundService::class.java)
            if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
                context.startForegroundService(intent)
            } else {
                context.startService(intent)
            }
        }
    }

    private val handler = Handler(Looper.getMainLooper())
    private var isRunning = false

    // 保活任务 - 定期检查服务状态
    private val keepAliveRunnable = object : Runnable {
        override fun run() {
            if (!isRunning) return

            // 检查无障碍服务状态
            if (!GuardAccessibilityService.isRunning) {
                Log.w(TAG, "无障碍服务未运行，尝试恢复")
                // 发送通知提醒用户
                showAccessibilityWarning()
            }

            // 检查AppBlockerService
            if (!isServiceRunning(AppBlockerService::class.java)) {
                Log.w(TAG, "AppBlockerService未运行，重新启动")
                AppBlockerService.startService(this@GuardForegroundService)
            }

            handler.postDelayed(this, 5000) // 每5秒检查一次
        }
    }

    override fun onCreate() {
        super.onCreate()
        isRunning = true

        // 启动其他服务
        startAllServices()

        // 启动保活检查
        handler.post(keepAliveRunnable)
    }

    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        createNotificationChannel()
        startForeground(NOTIFICATION_ID, createPersistentNotification())

        // MIUI优化：使用START_STICKY确保服务重启
        return START_STICKY
    }

    /**
     * 创建持久通知 - MIUI不容易杀死
     */
    private fun createPersistentNotification(): Notification {
        return NotificationCompat.Builder(this, CHANNEL_ID)
            .setContentTitle("儿童手机守护运行中")
            .setContentText("正在保护孩子免受不良应用影响")
            .setSmallIcon(android.R.drawable.ic_lock_idle_lock)
            .setOngoing(true) // 持续通知
            .setPriority(NotificationCompat.PRIORITY_HIGH) // 高优先级
            .setCategory(NotificationCompat.CATEGORY_SERVICE)
            .setVisibility(NotificationCompat.VISIBILITY_PUBLIC)
            .build()
    }

    /**
     * 检查服务是否运行
     */
    private fun isServiceRunning(serviceClass: Class<*>): Boolean {
        val manager = getSystemService(Context.ACTIVITY_SERVICE) as ActivityManager
        for (service in manager.getRunningServices(Integer.MAX_VALUE)) {
            if (serviceClass.name == service.service.className) {
                return true
            }
        }
        return false
    }

    override fun onDestroy() {
        super.onDestroy()
        isRunning = false
        handler.removeCallbacks(keepAliveRunnable)

        // MIUI优化：服务被杀时尝试重启
        val broadcastIntent = Intent("com.kidsphoneguard.RESTART_SERVICE")
        sendBroadcast(broadcastIntent)
    }
}
```

### 3.3 增强无障碍服务稳定性

修改`GuardAccessibilityService.kt`中的事件处理：

```kotlin
/**
 * 核心无障碍服务 - MIUI优化版
 */
class GuardAccessibilityService : AccessibilityService() {

    companion object {
        @Volatile
        var isRunning: Boolean = false
            private set

        // MIUI系统包名
        private val miuiPackages = setOf(
            "com.miui.home",
            "com.miui.securitycenter",
            "com.miui.powerkeeper",
            "com.miui.cleanmaster",
            "com.miui.bugreport"
        )
    }

    override fun onServiceConnected() {
        super.onServiceConnected()
        isRunning = true

        // MIUI优化：配置服务以接收更多事件
        serviceInfo = serviceInfo.apply {
            // 确保接收所有窗口事件
            eventTypes = AccessibilityEvent.TYPE_WINDOW_STATE_CHANGED or
                    AccessibilityEvent.TYPE_WINDOWS_CHANGED or
                    AccessibilityEvent.TYPE_WINDOW_CONTENT_CHANGED or
                    AccessibilityEvent.TYPE_VIEW_CLICKED

            // 设置反馈类型
            feedbackType = AccessibilityServiceInfo.FEEDBACK_GENERIC

            // 设置标志
            flags = AccessibilityServiceInfo.FLAG_RETRIEVE_INTERACTIVE_WINDOWS or
                    AccessibilityServiceInfo.FLAG_INCLUDE_NOT_IMPORTANT_VIEWS
        }

        Log.d(TAG, "服务已连接，MIUI优化已启用")
    }

    override fun onAccessibilityEvent(event: AccessibilityEvent) {
        // MIUI优化：增加日志以便调试
        if (BuildConfig.DEBUG) {
            Log.d(TAG, "收到事件: ${event.eventType}, 包名: ${event.packageName}")
        }

        when (event.eventType) {
            AccessibilityEvent.TYPE_WINDOW_STATE_CHANGED -> {
                handleWindowStateChanged(event)
            }
            AccessibilityEvent.TYPE_WINDOWS_CHANGED -> {
                handleWindowsChanged(event)
            }
            AccessibilityEvent.TYPE_WINDOW_CONTENT_CHANGED -> {
                // MIUI可能需要更频繁的内容检查
                handleWindowContentChanged(event)
            }
        }
    }

    /**
     * 处理窗口状态改变 - MIUI优化
     */
    private fun handleWindowStateChanged(event: AccessibilityEvent) {
        val packageName = event.packageName?.toString() ?: return

        // 跳过系统应用
        if (isSystemPackage(packageName)) {
            return
        }

        // MIUI优化：立即检查并拦截
        serviceScope.launch {
            val shouldBlock = checkShouldBlock(packageName)
            if (shouldBlock) {
                // 立即执行拦截
                performBlock(packageName)
            }
        }
    }

    /**
     * MIUI优化：更激进的拦截策略
     */
    private fun performBlock(packageName: String) {
        // 立即执行返回
        performGlobalAction(GLOBAL_ACTION_BACK)

        // MIUI优化：多次执行确保拦截成功
        handler.postDelayed({ performGlobalAction(GLOBAL_ACTION_BACK) }, 10)
        handler.postDelayed({ performGlobalAction(GLOBAL_ACTION_BACK) }, 50)
        handler.postDelayed({ performGlobalAction(GLOBAL_ACTION_HOME) }, 100)
        handler.postDelayed({ performGlobalAction(GLOBAL_ACTION_BACK) }, 200)
        handler.postDelayed({ performGlobalAction(GLOBAL_ACTION_HOME) }, 300)

        // 显示覆盖层
        OverlayService.showOverlay(this, packageName, "")
    }

    /**
     * 检查是否为系统包
     */
    private fun isSystemPackage(packageName: String): Boolean {
        return systemUIPackages.any { packageName.contains(it) } ||
               miuiPackages.any { packageName.contains(it) }
    }
}
```

### 3.4 添加服务重启机制

创建`ServiceRestartReceiver.kt`：

```kotlin
/**
 * 服务重启广播接收器
 * 当服务被系统杀死后自动重启
 */
class ServiceRestartReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent?) {
        when (intent?.action) {
            Intent.ACTION_BOOT_COMPLETED -> {
                // 开机启动
                startAllServices(context)
            }
            "com.kidsphoneguard.RESTART_SERVICE" -> {
                // 服务被杀后重启
                startAllServices(context)
            }
            Intent.ACTION_SCREEN_ON -> {
                // 屏幕亮起时检查服务
                if (!GuardAccessibilityService.isRunning) {
                    showAccessibilityNotification(context)
                }
            }
        }
    }

    private fun startAllServices(context: Context) {
        // 启动前台服务
        GuardForegroundService.startService(context)

        // 启动拦截服务
        AppBlockerService.startService(context)

        Log.d(TAG, "服务已重启")
    }

    private fun showAccessibilityNotification(context: Context) {
        // 显示通知提醒用户开启无障碍服务
        val notificationManager = context.getSystemService(Context.NOTIFICATION_SERVICE) as NotificationManager

        val channel = NotificationChannel(
            "accessibility_channel",
            "无障碍服务",
            NotificationManager.IMPORTANCE_HIGH
        )
        notificationManager.createNotificationChannel(channel)

        val intent = Intent(Settings.ACTION_ACCESSIBILITY_SETTINGS)
        val pendingIntent = PendingIntent.getActivity(
            context, 0, intent, PendingIntent.FLAG_IMMUTABLE
        )

        val notification = NotificationCompat.Builder(context, "accessibility_channel")
            .setContentTitle("儿童手机守护需要重新开启")
            .setContentText("无障碍服务被关闭，请点击重新开启")
            .setSmallIcon(android.R.drawable.ic_dialog_alert)
            .setContentIntent(pendingIntent)
            .setAutoCancel(true)
            .build()

        notificationManager.notify(1004, notification)
    }
}
```

## 六、常见问题

### Q1: 设置了所有权限还是无法拦截？

**可能原因**：
- MIUI版本过新，有更严格的限制
- 无障碍服务被系统暂停

**解决方案**：
1. 尝试重启手机
2. 重新开启无障碍服务
3. 检查MIUI是否有"应用行为记录"限制

### Q2: 拦截有延迟或时灵时不灵？

**可能原因**：
- 后台服务被系统限制
- 电池优化未关闭

**解决方案**：
1. 确保设置省电策略为"无限制"
2. 在最近任务中锁定应用
3. 关闭MIUI的"省电模式"

### Q3: 覆盖层不显示？

**可能原因**：
- 悬浮窗权限未开启
- MIUI限制了悬浮窗显示

**解决方案**：
1. 检查悬浮窗权限
2. 尝试关闭MIUI的"悬浮窗权限管理"
3. 重启无障碍服务

## 七、总结

### MIUI拦截失效的核心原因

1. **后台限制**：MIUI的省电策略会杀死后台服务
2. **服务限制**：无障碍服务在后台可能被暂停
3. **权限管理**：MIUI有额外的权限控制层
