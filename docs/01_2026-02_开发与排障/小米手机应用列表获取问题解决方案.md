# 小米手机应用列表获取问题完整解决方案

## 问题概述

**现象**：在小米手机上添加应用规则时，无法获取应用列表或列表为空

**根本原因**：
1. **Android 11+ 包可见性限制** - Google在Android 11引入了更严格的包可见性控制
2. **MIUI权限管理** - 小米MIUI系统对应用获取其他应用信息有额外限制
3. **QUERY_ALL_PACKAGES权限** - Android 11+需要显式声明此权限
4. **后台权限限制** - MIUI可能限制了应用的后台活动

```
□ 检查Android版本（设置 → 我的设备 → Android版本）
□ 检查MIUI版本（设置 → 我的设备 → MIUI版本）
```

## 二、解决方案

### 2.1 方案一：AndroidManifest.xml权限声明（已实施）

已在`AndroidManifest.xml`中添加：

```xml
<!-- Android 11+ 需要此权限才能查询所有应用 -->
<uses-permission android:name="android.permission.QUERY_ALL_PACKAGES" />
```

**注意**：从Android 11开始，Google Play要求申请此权限的应用必须通过审核，说明为什么需要此权限。

### 2.2 方案二：改进应用扫描逻辑（已实施）

已重写`AppScanner.kt`，采用双策略方案：

```kotlin
/**
 * 策略1: 使用queryIntentActivities（推荐，兼容性最好）
 * 通过查询可启动的Activity来获取应用列表
 * 此方法不受包可见性限制影响
 */
val launcherIntent = Intent(Intent.ACTION_MAIN).apply {
    addCategory(Intent.CATEGORY_LAUNCHER)
}
val resolveInfos = packageManager.queryIntentActivities(launcherIntent, 0)

/**
 * 策略2: 使用getInstalledPackages（备选方案）
 * 当策略1获取的应用过少时启用
 * 需要QUERY_ALL_PACKAGES权限
 */
if (apps.size < 5) {
    apps.addAll(getAppsFromInstalledPackages(context, packageManager, includeSystemApps))
}
```

### 2.3 方案三：MIUI特定设置（用户操作）

#### 步骤1: 开启"获取应用列表"权限

```
设置路径：
设置 → 应用设置 → 应用管理 → 儿童手机守护 → 权限管理 → 获取应用列表 → 允许

MIUI 12/13/14 路径：
设置 → 应用设置 → 权限 → 权限管理 → 获取应用列表 → 找到"儿童手机守护" → 允许
```

#### 步骤2: 允许自启动和后台运行

```
设置路径：
设置 → 应用设置 → 应用管理 → 儿童手机守护 → 省电策略 → 无限制

或：
手机管家 → 应用管理 → 权限 → 自启动管理 → 允许"儿童手机守护"自启动

或：
设置 → 电池与性能 → 应用智能省电 → 儿童手机守护 → 无限制
```

#### 步骤3: 锁定应用（防止被清理）

```
操作步骤：
1. 打开最近任务（多任务界面）
2. 找到"儿童手机守护"
3. 长按应用卡片或点击锁定图标
4. 确保应用卡片右上角显示锁形图标
```

### 2.4 方案四：添加queries声明（Android 11+推荐）

在`AndroidManifest.xml`的`<manifest>`标签内添加：

```xml
<!-- 如果需要查询特定应用，可以声明queries -->
<queries>
    <!-- 查询所有可启动的应用 -->
    <intent>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent>
    
    <!-- 如果需要查询特定包名 -->
    <package android:name="com.example.app" />
</queries>
```

**注意**：添加queries声明后，可以不使用QUERY_ALL_PACKAGES权限也能获取部分应用信息。

## 三、各品牌手机特定解决方案

### 3.1 小米手机（MIUI）

| 问题 | 解决方案 |
|-----|---------|
| 无法获取应用列表 | 开启"获取应用列表"权限 |
| 应用被系统杀死 | 设置省电策略为"无限制" |
| 后台服务被清理 | 在最近任务中锁定应用 |
| 权限自动关闭 | 关闭MIUI的权限自动重置功能 |

**MIUI版本特定路径**：

```
MIUI 12/12.5:
设置 → 应用设置 → 应用管理 → [应用名] → 权限管理 → 获取应用列表

MIUI 13:
设置 → 隐私保护 → 保护隐私 → 应用权限设置 → 获取应用列表

MIUI 14:
设置 → 隐私与安全 → 应用权限 → 获取应用列表
```

### 3.2 华为手机（HarmonyOS/EMUI）

| 问题 | 解决方案 |
|-----|---------|
| 无法获取应用列表 | 开启"读取已安装应用列表"权限 |
| 应用被系统杀死 | 设置电池优化为"不允许" |
| 后台服务被限制 | 开启"允许后台活动" |

**设置路径**：

```
设置 → 应用和服务 → 应用管理 → 儿童手机守护 → 权限 → 读取已安装应用列表 → 允许

设置 → 电池 → 应用启动管理 → 儿童手机守护 → 手动管理 → 
  □ 允许自启动
  □ 允许关联启动
  □ 允许后台活动
```

### 3.3 OPPO手机（ColorOS）

| 问题 | 解决方案 |
|-----|---------|
| 无法获取应用列表 | 开启"读取应用列表"权限 |
| 应用被系统杀死 | 设置电池优化为"不优化" |
| 后台服务被清理 | 在最近任务中锁定应用 |

**设置路径**：

```
设置 → 应用管理 → 应用列表 → 儿童手机守护 → 权限管理 → 读取应用列表 → 允许

设置 → 电池 → 应用耗电管理 → 儿童手机守护 → 
  □ 允许完全后台行为
  □ 允许应用自启动
```

### 3.4 vivo手机（Funtouch OS/OriginOS）

| 问题 | 解决方案 |
|-----|---------|
| 无法获取应用列表 | 开启"读取已安装应用信息"权限 |
| 应用被系统杀死 | 设置后台高耗电允许 |
| 后台服务被限制 | 开启自启动权限 |

**设置路径**：

```
设置 → 应用与权限 → 权限管理 → 读取已安装应用信息 → 儿童手机守护 → 允许

设置 → 电池 → 后台耗电管理 → 儿童手机守护 → 允许后台高耗电

i管家 → 应用管理 → 权限管理 → 自启动 → 允许儿童手机守护自启动
```

### 3.5 三星手机（One UI）

| 问题 | 解决方案 |
|-----|---------|
| 无法获取应用列表 | 开启"查询所有软件包"权限 |
| 应用被系统杀死 | 关闭电池优化 |
| 后台服务被限制 | 允许后台活动 |

**设置路径**：

```
设置 → 应用程序 → 儿童手机守护 → 权限 → 其他权限 → 查询所有软件包 → 允许

设置 → 电池和设备维护 → 电池 → 后台使用限制 → 深度休眠应用程序 → 
确保"儿童手机守护"不在列表中

设置 → 应用程序 → 儿童手机守护 → 电池 → 允许后台活动
```

## 六、预防措施

### 6.1 应用启动时检查

```kotlin
class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        
        // 检查是否能获取应用列表
        if (!AppScanner.canGetAppList(this)) {
            // 显示权限引导对话框
            showPermissionGuideDialog()
        }
    }
}
```

在应用描述和首次启动时明确说明：

```
"儿童手机守护"需要"获取应用列表"权限才能：
1. 显示设备上已安装的应用
2. 允许家长选择需要管控的应用
3. 执行应用使用时间管理

我们不会收集或上传您的应用列表信息，所有数据仅保存在本地设备上。
```

## 七、总结

### 核心解决方案

1. **代码层面**：使用`queryIntentActivities`替代`getInstalledPackages`，兼容性更好
2. **权限层面**：声明`QUERY_ALL_PACKAGES`权限（Android 11+）
3. **用户引导**：提供详细的权限开启指引，特别是MIUI等定制系统
4. **诊断工具**：添加权限状态检查功能，便于排查问题

- [ ] 在小米/华为/OPPO/vivo手机上测试通过
