## 2. B3问题定义（仅事实）

### 2.1 当前代码行为
- `UsageTrackingManager` 在“超限”时：
  - 若无障碍服务运行中：发送 BLOCK_APP 广播；
  - 若无障碍服务未运行：仅显示 Overlay。
- 对应代码：
  - `app/src/main/java/com/kidsphoneguard/service/UsageTrackingManager.kt`（144-150）

### 2.2 广播拦截链路
- BLOCK_APP 广播由 `GuardAccessibilityService` 接收并执行 `enforceBlock(...)`。
- 对应代码：
  - `app/src/main/java/com/kidsphoneguard/service/GuardAccessibilityService.kt`（57-67, 108-112）

### 2.3 实质含义
- 当无障碍不可用时，系统级强制返回/强制终止链路不可用，当前只能依赖 Overlay 作为限制手段。

## 3. B3是否修复的取舍依据

### 3.1 不修复的后果
- 超限后限制能力下降为“仅遮罩层”。
- 在部分机型或场景下，用户可能仍可绕过遮罩继续使用目标应用。

### 3.2 修复可达到的目的
- 无障碍不可用场景下，提升超限后的实际阻断成功率。
- 减少“规则已超限但仍可使用”的体验分裂。

### 3.3 修复可能带来的新问题
- 增加复杂度与维护成本（更多分支与状态）。
- 可能引入误触发（例如短时切换前后台时的误打断）。
- 对不同 ROM 行为差异更敏感，兼容测试成本变高。

## 7. 第二轮体验修正（基于用户回归反馈）

### 7.1 反馈事实
- 用户反馈：仍存在多轮闪烁；遮蔽层颜色刺眼；遮蔽层按钮无明显价值。
- 日志观察：`GuardAccessibilityService` 高频出现 `com.android.systemui 在白名单中，跳过锁定`。

### 7.2 根因判断与处理决策
- 决策 D1：保留白名单跳过，但对 `com.android.systemui` 不触发遮蔽层隐藏，降低系统UI事件引起的闪烁。
- 决策 D2：移除遮蔽层按钮，维持纯提示遮蔽，减少误触与认知负担。
- 决策 D3：将遮蔽层底色改为低刺激深色半透明，降低视觉压迫。

### 7.3 实际修改文件
- `app/src/main/java/com/kidsphoneguard/service/GuardAccessibilityService.kt`
- `app/src/main/java/com/kidsphoneguard/service/OverlayService.kt`

### 7.4 实际动作
- 在白名单分支中增加 `com.android.systemui` 特判，不因系统UI事件关闭遮蔽层。
- 移除 Overlay 按钮控件与对应点击逻辑。
- 将遮蔽层底色从高刺激色改为深色半透明（`#B3121212`）。

### 7.5 按钮必要性评估结论
- 当前链路中，拦截执行依赖无障碍返回与任务清理，按钮不参与拦截闭环。
- 用户目标是“自动拦截，无需手动确认”，按钮属于冗余交互。
- 结论：在当前产品策略下按钮可移除，保留纯遮蔽提示更简洁。

## 10. 第五轮紧急修正（重应用拦截失效）

### 10.1 反馈事实
- 用户反馈：`com.supercell.clashofclans` 出现遮蔽层后仍持续运行，未被真正拦截。

### 10.2 日志与代码判断
- 日志显示阻塞决策命中：`TIME_WINDOW_BLOCKED`。
- 日志显示 `forceStopPackage` 被系统权限拒绝（`requires android.permission.FORCE_STOP_PACKAGES`），导致“杀进程”路径在普通权限下不可用。
- 冷却期策略会让后续短时间拦截被跳过，放大重应用未退出问题。

### 10.3 修正决策
- 决策 G1：保留现有链路，增加“冷却期但仍在前台则继续拦截”的兜底分支。
- 决策 G2：对仍在前台的目标包执行二次兜底导航（Back + 条件 Home），确保重应用被切离前台。

### 10.4 实际修改文件
- `app/src/main/java/com/kidsphoneguard/service/GuardAccessibilityService.kt`

### 10.5 实际动作
- 新增 `isTargetPackageActive(packageName)` 前台判断。
- 冷却期判断改为：目标包若仍在前台，不再跳过本轮拦截。
- 新增 `tryFallbackNavigation(packageName)`：仅在目标包仍前台时执行兜底返回与回桌面。
