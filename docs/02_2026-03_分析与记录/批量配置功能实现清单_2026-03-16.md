# 批量配置功能实现清单（V1 + V2）

## 目标

- 将应用按“已配置/未配置”分层管理（系统白名单除外）
- 批量配置默认仅面向未配置应用，避免同一应用出现规则冲突
- 保持同一应用任一时刻仅有一个生效规则
- 增强 LIMIT 规则，支持仅限时长、仅限时段、时长+时段三种模式
- 白名单匹配逻辑保持现状，不调整 `contains` 判断

## 范围说明

- V1：冲突规避主链路（默认批量仅配置未配置应用）
- V2：引入 LIMIT 模式扩展 + 批量重配置模式（可选开启）

## 设计原则

- 入口即约束：默认只给未配置应用开放批量配置入口
- 单应用单规则：`ALLOW/BLOCK/LIMIT` 三选一
- 批量操作可回执：成功数、跳过数、跳过原因可见
- 数据写入有保护：提交前后双重校验 + 批量原子写入

## 实施项

### V1：默认仅配置未配置应用

1. 数据层
   - 在 DAO 增加“已配置包名查询”能力
   - 在仓库层增加批量提交入口，统一处理预校验与写入

2. 批量候选过滤
   - 应用列表先过滤系统白名单
   - 再过滤已配置应用
   - 最终仅展示未配置应用

3. 提交流程
   - 提交前校验目标应用是否仍为未配置
   - 若存在并发变更导致已配置，跳过并记录原因
   - 提交结果回执：成功/跳过统计 + 冲突明细

4. 交互优化
   - 批量弹窗展示统计：可配置总数、已过滤数、当前选中数
   - 提交中禁用按钮，防重复提交
   - 提交完成弹窗展示结果摘要

### V2：可选批量重配置 + LIMIT 三模式

1. 规则模型扩展
   - 在 `AppRule` 增加 `limitMode`
   - 模式值：`BOTH`、`DURATION_ONLY`、`WINDOW_ONLY`

2. 批量配置模式
   - 新增开关：`仅未配置（默认）` / `重配置已配置`
   - 默认保持简化策略，仅未配置可选
   - 开启重配置后，允许覆盖既有规则

3. LIMIT 表单交互
   - 选择 LIMIT 后先选模式
   - 按模式显示输入项：
     - 仅时长：只填分钟
     - 仅时段：只填时段
     - 双模式：同时填分钟和时段

4. 决策逻辑
   - `DURATION_ONLY`：仅执行时长判定
   - `WINDOW_ONLY`：仅执行时段判定
   - `BOTH`：任一条件触发即拦截

## 测试清单

### 单元测试

- 未配置过滤正确（已配置与系统白名单均不进入默认可选清单）
- 批量提交默认模式仅写入未配置应用
- 批量重配置模式可覆盖旧规则但最终仍单规则
- LIMIT 三模式判定准确

### 集成测试

- 批量提交事务一致性
- 并发状态变化下的二次校验与跳过策略
- 回执统计与实际落库一致

### 验证命令

- `.\gradlew.bat compileDebugKotlin`
- `.\gradlew.bat :app:lint`
- `.\gradlew.bat test`

## 验收标准

- 默认批量页只出现未配置应用
- 已配置应用不会在默认模式被再次批量配置
- 开启重配置模式后可覆盖，但最终每个应用仅一个规则
- LIMIT 三模式配置与拦截行为一致
- 白名单匹配逻辑无变更
