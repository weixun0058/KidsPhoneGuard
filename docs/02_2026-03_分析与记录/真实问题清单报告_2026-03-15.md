# 真实问题清单报告（基于当前代码）

**生成日期**：2026-03-15  
**核验范围**：`app/src/main/java`、`app/src/main/AndroidManifest.xml`、`app/src/main/res/xml/accessibility_service_config.xml`  
**核验原则**：仅保留“当前代码可直接证实”的问题，剔除已过时结论与推测性描述。

---

## 一、P0（必须优先修复）

### 1. 白名单匹配存在包名伪装绕过风险
- 现状：白名单使用 `contains()` 匹配包名。
- 风险：恶意包名可通过包含白名单字符串绕过限制。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/utils/WhitelistManager.kt:264-266`

### 2. 密码明文存储且存在默认密码
- 现状：密码直接写入 `SharedPreferences`，默认密码硬编码。
- 风险：设备被获取数据后可直接读取，默认口令降低破解门槛。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/utils/PasswordManager.kt:17`
  - `app/src/main/java/com/kidsphoneguard/utils/PasswordManager.kt:35-45`

### 3. 时长超限时，缺少“无障碍不可用”兜底强制拦截
- 现状：超限后仅在无障碍服务可用时发送拦截广播；不可用时只显示覆盖层。
- 风险：覆盖层可被规避后继续使用应用。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/service/UsageTrackingManager.kt:144-150`

## 二、P1（高优先级）

### 1. 广播保护不完整（版本相关风险）
- 现状：发送广播未附带权限；接收端仅在 Android 13+ 使用 `RECEIVER_NOT_EXPORTED`。
- 风险：低版本系统上广播被外部构造触发的概率更高。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/utils/BroadcastPermissionHelper.kt:24-27`
  - `app/src/main/java/com/kidsphoneguard/utils/BroadcastPermissionHelper.kt:42-47`

### 2. KILL 广播命名不一致
- 现状：一处常量为 `com.kidsphoneguard.action.KILL_APP`，另一处发送 `"ACTION_KILL_APP"`。
- 风险：行为分裂、联动失效、后续维护混乱。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/utils/BroadcastPermissionHelper.kt:13`
  - `app/src/main/java/com/kidsphoneguard/service/AppKillerService.kt:263`

### 3. 无障碍监听范围过宽
- 现状：代码监听窗口内容变化、辅助阅读上下文等事件；配置使用 `typeAllMask`。
- 风险：采集面过大，增加敏感信息暴露与合规风险。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/service/GuardAccessibilityService.kt:136-142`
  - `app/src/main/res/xml/accessibility_service_config.xml:3`

### 4. Overlay 静态状态集中管理，存在竞态条件
- 现状：`overlayView`、`isShowing`、`currentPackageName` 在 `companion object` 中全局共享。
- 风险：多入口并发显示/隐藏时可能闪烁或状态错乱。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/service/OverlayService.kt:26-30`

### 5. 数据库升级策略为 destructive migration
- 现状：启用 `fallbackToDestructiveMigration()`。
- 风险：未来升级版本时用户规则与统计数据可能丢失。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/data/db/AppDatabase.kt:40`

## 三、P2（中优先级）

### 1. 强杀策略包含反射与 shell 命令，兼容性/合规性风险高
- 现状：使用 `forceStopPackage` 反射与 `am force-stop`。
- 风险：ROM 差异导致不稳定，商店审核与系统限制风险增加。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/service/GuardAccessibilityService.kt:319-327`

### 2. 全局锁状态存在双来源
- 现状：同时读取 `SettingsManager` 与 `AppRule.isGlobalLocked`。
- 风险：状态一致性与排障复杂度上升。
- 证据：
  - `app/src/main/java/com/kidsphoneguard/engine/LockDecisionEngine.kt:39-44`
